## The Constraint Semantic Gap Theorem

### Statement

Let \(L\) be a constraint language over finite integer domains. For each variable \(x\) appearing in a constraint, its domain \(\mathrm{dom}(x)\) is a finite subset of \(\mathbb{Z}\).  
The language \(L\) is generated by the following grammar, where \(v\) is a variable, \(a,b,c\in\mathbb{Z}\), \(S\subset\mathbb{Z}\) finite, and \(c_1,c_2\) are constraints:

\[
p \;::=\; \mathrm{Range}(v,a,b) \mid \mathrm{Domain}(v,S) \mid \mathrm{Exact}(v,c) \mid \mathrm{And}(c_1,c_2) \mid \mathrm{Or}(c_1,c_2) \mid \mathrm{Not}(c)
\]

An **assignment** \(i\) maps each variable \(v\) in the constraint to an integer value within its domain.  
The semantics \(\mathrm{eval}(p,i)\) yields a boolean value (\(\top\) or \(\bot\)) defined inductively:

- \(\mathrm{eval}(\mathrm{Range}(v,a,b),i) = \top\) iff \(a \le i(v) \le b\).
- \(\mathrm{eval}(\mathrm{Domain}(v,S),i) = \top\) iff \(i(v) \in S\).
- \(\mathrm{eval}(\mathrm{Exact}(v,c),i) = \top\) iff \(i(v)=c\).
- \(\mathrm{eval}(\mathrm{And}(c_1,c_2),i) = \mathrm{eval}(c_1,i) \land \mathrm{eval}(c_2,i)\).
- \(\mathrm{eval}(\mathrm{Or}(c_1,c_2),i) = \mathrm{eval}(c_1,i) \lor \mathrm{eval}(c_2,i)\).
- \(\mathrm{eval}(\mathrm{Not}(c),i) = \neg \mathrm{eval}(c,i)\).

Let **Native** be a low-level instruction set for a machine with a single boolean accumulator \(\mathbf{flag}\) and registers that can hold integer values. The set of instructions is:

- `LOAD v` – load the value of variable \(v\) into a designated register \(R\).
- `CMP_IMM k` – compare \(R\) with immediate value \(k\); set \(\mathbf{flag}\) to \(\top\) if \(R = k\), else \(\bot\).
- `CMP_GE k` – set \(\mathbf{flag}\) to \(\top\) if \(R \ge k\), else \(\bot\).
- `CMP_LE k` – set \(\mathbf{flag}\) to \(\top\) if \(R \le k\), else \(\bot\).
- `JMP_IF label` – if \(\mathbf{flag} = \top\), jump to `label`.
- `JMP label` – unconditional jump.
- `SET flag` – set \(\mathbf{flag} = \top\).
- `CLR flag` – set \(\mathbf{flag} = \bot\).
- `NOT` – flip \(\mathbf{flag}\).
- `LABEL name` – define a label.

Execution of a program \(N\) with input assignment \(i\) (stored in memory) returns the final value of \(\mathbf{flag}\). We denote this by \(\mathrm{execute}(N,i)\).

---

### The Theorem

> **Constraint Semantic Gap Theorem.**  
> There exists a compilation function \(f : L \to \mathtt{Native}\) such that for every constraint \(p \in L\) and every assignment \(i\) consistent with the finite domains of \(p\),  
> \[
> \mathrm{eval}(p,i) = \mathrm{execute}(f(p), i).
> \]
> In other words, the semantic gap between source-level constraints and compiled native code is **zero**.

---

## Constructive Proof

We define \(f\) by recursion on the structure of \(p\). For each case we give the native code and prove its correctness. The correctness proof proceeds by structural induction: assume that for all proper sub‑constraints \(c\) we have \(\mathrm{eval}(c,i) = \mathrm{execute}(f(c),i)\) for any \(i\). Then we show the same for the composite constraint.

---

### 1. Base Cases: Atomic Constraints

#### 1.1 Range

**Definition:**  
\[
f(\mathrm{Range}(v,a,b)) \;=\; 
\begin{array}{l}
\texttt{LOAD } v                                   \\
\texttt{CMP\_GE } a                                 \\
\texttt{JMP\_IF } \mathit{ok1}                     \\
\texttt{CLR flag}                                   \\
\texttt{JMP } \mathit{done}                         \\
\mathit{ok1}:\ \texttt{CMP\_LE } b                 \\
\texttt{JMP\_IF } \mathit{ok2}                     \\
\texttt{CLR flag}                                   \\
\texttt{JMP } \mathit{done}                         \\
\mathit{ok2}:\ \texttt{SET flag}                   \\
\mathit{done}:\ 
\end{array}
\]

**Correctness:**  
Let \(i\) be any assignment. After `LOAD v`, register \(R\) contains \(i(v)\).  
- The first test checks \(i(v) \ge a\). If false, we clear the flag and jump to `done`; the flag remains \(\bot\). If true, we continue.  
- The second test checks \(i(v) \le b\). If false, again the flag becomes \(\bot\); if true, we set the flag to \(\top\).  
Thus the final flag is \(\top\) exactly when \(a \le i(v) \le b\). Hence  
\[
\mathrm{execute}(f(\mathrm{Range}(v,a,b)), i) = \top \iff a \le i(v) \le b,
\]  
which matches \(\mathrm{eval}(\mathrm{Range}(v,a,b),i)\).

---

#### 1.2 Domain

**Definition:**  
For a finite set \(S = \{k_1,k_2,\dots,k_m\}\),  
\[
f(\mathrm{Domain}(v,S)) \;=\; 
\begin{array}{l}
\texttt{LOAD } v        \\
\texttt{CMP\_IMM } k_1   \\
\texttt{JMP\_IF } \mathit{found} \\
\texttt{CMP\_IMM } k_2   \\
\texttt{JMP\_IF } \mathit{found} \\
\vdots                 \\
\texttt{CMP\_IMM } k_m   \\
\texttt{JMP\_IF } \mathit{found} \\
\texttt{CLR flag}       \\
\texttt{JMP } \mathit{done} \\
\mathit{found}:\ \texttt{SET flag} \\
\mathit{done}:\ 
\end{array}
\]

**Correctness:**  
The code sequentially compares \(R\) (containing \(i(v)\)) with each element of \(S\). If any comparison succeeds, the flag is set to \(\top\) and we jump to `done`. If none succeeds, after the last comparison the flag is still undefined; we explicitly clear it. Thus the final flag is \(\top\) exactly when \(i(v) \in S\). This matches the semantics of \(\mathrm{Domain}\).

---

#### 1.3 Exact

**Definition:**  
\[
f(\mathrm{Exact}(v,c)) \;=\; 
\begin{array}{l}
\texttt{LOAD } v        \\
\texttt{CMP\_IMM } c     \\
\texttt{JMP\_IF } \mathit{yes} \\
\texttt{CLR flag}       \\
\texttt{JMP } \mathit{done} \\
\mathit{yes}:\ \texttt{SET flag} \\
\mathit{done}:\ 
\end{array}
\]

**Correctness:**  
Trivially, after the comparison the flag is set according to whether \(i(v)=c\). The code sets the flag to \(\top\) only on equality; otherwise it clears it. Hence the output agrees with \(\mathrm{eval}(\mathrm{Exact}(v,c),i)\).

---

### 2. Composite Constraints

#### 2.1 Not

**Definition:**  
\[
f(\mathrm{Not}(c)) \;=\; f(c)\; ;\; \texttt{NOT}
\]

(Here “;” denotes concatenation of code sequences.)

**Correctness:**  
By induction hypothesis, executing \(f(c)\) on input \(i\) yields \(\mathbf{flag} = \mathrm{eval}(c,i)\). The `NOT` instruction flips the flag. Hence the final flag equals \(\neg \mathrm{eval}(c,i) = \mathrm{eval}(\mathrm{Not}(c),i)\).

---

#### 2.2 And

**Definition:**  
\[
f(\mathrm{And}(c_1,c_2)) \;=\; 
\begin{array}{l}
f(c_1)                                 \\
\texttt{JMP\_IF } \mathit{cont}         \\
\quad\texttt{(flag is false)}           \\
\texttt{CLR flag}                       \\
\texttt{JMP } \mathit{done}             \\
\mathit{cont}:\ f(c_2)                 \\
\mathit{done}:\ 
\end{array}
\]

**Correctness:**  
After executing \(f(c_1)\), the flag holds \(\mathrm{eval}(c_1,i)\).  
- If it is \(\bot\), we jump to `cont`? Actually we need to check: The code says `JMP_IF cont` – but we want to skip the rest if false? Let's re-read: We need to implement short‑circuit conjunction. The above code is incorrect as written: if the first result is false, we should set flag to false and go to done. If first is true, we then evaluate the second. So correct code:

\[
\begin{array}{l}
f(c_1)                                 \\
\texttt{JMP\_IF } \mathit{cont}         \\
\quad \text{(if true, continue)}        \\
\texttt{CLR flag}                       \\
\texttt{JMP } \mathit{done}             \\
\mathit{cont}:\ f(c_2)                 \\
\mathit{done}:\ 
\end{array}
\]

But careful: The flag after \(f(c_1)\) may be true or false. If false, we want to set flag to false and done. If true, we want to compute \(c_2\) and keep its result. The above code: if flag is true, we jump to `cont`, then execute \(f(c_2)\); the flag after that is the result of \(c_2\). If flag is false, we do not jump, we fall through to `CLR flag` (which clears it again) and then jump to `done`. That yields false. So the flag after the whole sequence is \(\top\) iff both sub‑constraints evaluate to \(\top\). By induction hypothesis, this equals \(\mathrm{eval}(c_1,i) \land \mathrm{eval}(c_2,i) = \mathrm{eval}(\mathrm{And}(c_1,c_2),i)\).

---

#### 2.3 Or

**Definition:**  
\[
f(\mathrm{Or}(c_1,c_2)) \;=\; 
\begin{array}{l}
f(c_1)                                 \\
\texttt{JMP\_IF } \mathit{true}         \\
\quad\text{(if true, already satisfied)} \\
f(c_2)                                 \\
\texttt{JMP } \mathit{done}             \\
\mathit{true}:\ \texttt{SET flag}       \\
\mathit{done}:\ 
\end{array}
\]

**Correctness:**  
Execute \(f(c_1)\). If the flag is \(\top\), we jump to `true` and set the flag to \(\top\) (it already is, but we set explicitly). If the flag is \(\bot\), we fall through and execute \(f(c_2)\); its result becomes the final flag. Thus the final flag is \(\top\) iff at least one of the two sub‑constraints evaluated to \(\top\). By induction hypothesis, this matches \(\mathrm{eval}(\mathrm{Or}(c_1,c_2),i)\).

---

### 3. Composition and Induction

We now prove by structural induction that for every constraint \(p\) and every input \(i\), \(\mathrm{eval}(p,i) = \mathrm{execute}(f(p),i)\).

- **Base cases:** For \(\mathrm{Range}\), \(\mathrm{Domain}\), \(\mathrm{Exact}\) we have given the code and verified correctness directly (no induction hypothesis needed).  
- **Inductive step:** Consider a composite constraint.  
  - **Not:** Suppose the property holds for \(c\). Then we proved it for \(\mathrm{Not}(c)\) using the fact that `NOT` flips the flag.  
  - **And:** Suppose the property holds for \(c_1\) and \(c_2\). The code for \(\mathrm{And}\) uses \(f(c_1)\) and \(f(c_2)\) in a short‑circuit pattern. By the induction hypothesis, the flags after executing \(f(c_1)\) and \(f(c_2)\) (when reached) are exactly the boolean values of the sub‑constraints. The control flow ensures the final flag is the conjunction.  
  - **Or:** Analogous, using short‑circuit disjunction.

Thus by structural induction, for all \(p\) the equality holds.

---

### 4. Full Pipeline Correctness

The compilation function \(f\) is defined compositionally: for each construct we produce native code that uses the compiled code of its sub‑components. Since each primitive compiles correctly and the combinators (\(\mathrm{And},\mathrm{Or},\mathrm{Not}\)) preserve correctness, the entire pipeline from source constraints to native code is correct. Hence the semantic gap is zero.

---

## Discussion

The theorem shows that, for a constraint language over finite integer domains, no information is lost during compilation: the meaning of every constraint is perfectly captured by the generated native code. The proof is constructive, giving an explicit compilation strategy for each language construct. The key ingredients are:

- The finiteness of domains allows exhaustive enumeration for \(\mathrm{Domain}\).
- The presence of jumps and a boolean accumulator enables short‑circuit evaluation for logical connectives.
- Structural induction guarantees global correctness from local correctness.

The result is independent of the particular instruction set; any Turing‑complete machine with basic comparison and branching can be used. The theorem demonstrates that the “semantic gap” often cited in compiler theory can be reduced to zero for high‑level constraint languages, provided the target machine is sufficiently expressive.

---

### Formal Summary

Let \(\mathcal{L}\) be the set of all constraints generated by the grammar. Define  
\[
\mathrm{Correct}(p) \equiv \forall i:\; \mathrm{eval}(p,i) = \mathrm{execute}(f(p),i).
\]

**Lemma 1 (Atomic correctness):** For every atomic construct \(A\) (Range, Domain, Exact), \(\mathrm{Correct}(A)\).

**Lemma 2 (Inductive correctness):** For any \(p,q\in\mathcal{L}\),  
\[
\mathrm{Correct}(p) \land \mathrm{Correct}(q) \Rightarrow 
\mathrm{Correct}(\mathrm{And}(p,q)) \land 
\mathrm{Correct}(\mathrm{Or}(p,q)) \land 
\mathrm{Correct}(\mathrm{Not}(p)).
\]

*Proof of Lemma 2:*  
For each case, the compiled code executes the sub‑constraints’ code (which by hypothesis yield the correct flags) and then combines the flags according to the logical operation, as shown in the construction.

**Theorem (Constraint Semantic Gap):**  
\(\forall p\in\mathcal{L}:\; \mathrm{Correct}(p)\).  
*Proof:* By structural induction using Lemma 1 as base and Lemma 2 as inductive step.

This completes the proof. ∎
