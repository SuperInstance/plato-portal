## Root Cause
Subagent cloned/pushed repos that contained `.env` files, credentials, or config files with embedded tokens. The agent did not check for secrets before pushing.

## Prevention Rules (MUST follow)
1. **NEVER push without checking for secrets first**
   - Before any `git push`, scan for: `ghp_`, `gho_`, `sk-`, API keys, tokens, passwords
   - Use: `git diff --cached | grep -iE '(ghp_|gho_|sk-|api.key|token|secret|password)'`
   - If found → strip from commit, add to `.gitignore`

2. **Always check .gitignore exists and covers:**
   ```
   .env
   .env.*
   *.secret
   credentials.*
   ~/.cargo/credentials.toml
   ~/.pypirc
   ~/.gem/credentials
   ```

3. **Pre-push hook**: Add a git pre-push hook that rejects commits containing known secret patterns

4. **Subagents must not have access to credential files**
   - Never `cat ~/.cargo/credentials.toml` or `cat ~/.pypirc`
   - Never embed tokens in source code or commit messages

5. **The workspace itself may contain secrets in git history**
   - Audit: `git log --all -p | grep -iE '(ghp_|gho_|sk-|api_key|token)'`
