### 2. **constraint-theory-core** (2026-03-27) — Deterministic Manifold Snapping
**Genius idea:** Replace floating-point approximation with exact Pythagorean rational arithmetic — snap any continuous vector to the nearest (a/c, b/c) where a²+b²=c² holds in integers, not floats, using a KD-tree over a finite lattice of exact points.

**Ahead of its time because:** This was a MONTH before the 112× music conservation result. The Pythagorean manifold — deterministic, platform-independent, exact — was the mathematical seed that grew into the entire Conservation Spectral Framework. The 83 tests, zero deps, the Eisenstein lattice, deadband funnels, Laman rigidity — all of it started here. And the description is beautiful: "0.6² + 0.8² = 1.0000000000000002 — and you've been debugging this for years."

**Connects to:** The 112× result, deadband-rs, flux-vm-v3, constraint-dialect (MLIR), the entire spectral framework, the 20+ language SDK. Every theorem traces back to this crate.

**Do now:** Already published on crates.io. Should be **a standalone technical blog post** — "Why Your Floating Points Are Lying to You and What to Do About It." The deterministic multiplayer and robotics use cases are undersold. Could be a conference talk.

Released April 20 and then... got buried in the conservation sprint.

**Connects to:** The MUD Arena's forward simulations (t-minus-event), PLATO rooms where agents predict checkpoints, the flow-state architecture where agents simulate before acting. Dreamer IS the cognitive layer that makes FLUX flow-state possible.

**Do now:** **REVIVE IMMEDIATELY.** This is flow-state ready. Integrate with the conservation spectral framework — agents could dream along their Fiedler vector projection, exploring conservation-preserving futures. The dream memory becomes a spectral memory. This could be a standalone product: "Agent Dreaming as a Service." The MCTS + Laplacian combination is unexplored territory.

### 4. **sunset-ecosystem** (2026-05-21) — The Trinity Architecture
**Genius idea:** Every agent lives or dies by a trinity score = ethos × pathos × logos — a PRODUCT, not a weighted sum. If ANY dimension is zero, the agent sunsets. Retiring agents write epilogues, summaries, and onboarding letters for the next generation.

**Ahead of its time because:** The product-vs-sum insight is mathematically profound. Weighted sums let you trade off dimensions; products enforce hard constraints. This is the same principle as the conservation framework — you can't compensate for broken conservation with high performance elsewhere. The sunset documents (epilogue, summary, onboarding letter with continuation/cross-pollination/mutation variants) are the most elegant agent lifecycle design I've seen. Agents dying with dignity and seeding the future.

**Connects to:** Pareto-tournament (Pareto frontier for agent selection), the fleet's breeding and evolution cycles, PLATO's generational knowledge transfer. The ethos (hardware), pathos (human), logos (code) mapping is exactly what the spectral framework measures — conservation IS the product of alignment across dimensions.

**Do now:** **INTEGRATE into the spectral framework.** Trinity score should be expressed as a conservation ratio: the product of spectral conservation across three agent relationship graphs (hardware-human-code). The sunset documents should become PLATO tiles automatically. This is the lifecycle layer the fleet needs.

### 6. **actualizer-ai** (2026-04-10) — Reverse Actualization
**Genius idea:** Start from your desired future (100 years, 10 years, 1 year) and work BACKWARD to identify what matters this week — running six independent time-horizon analyses and cross-referencing them. Avoids rigid todo lists and artificial certainty.

**Ahead of its time because:** Most planning tools project forward. This inverts causality — the same mathematical inversion that conservation tomography uses (recover graph from spectral measurements = recover present from desired future). The multi-horizon cross-referencing is essentially spectral decomposition of a life plan.

**Connects to:** The Lucineer ecosystem, the fleet's goal management, and conceptually to the conservation framework's inverse problems. Reverse actualization IS inverse conservation: given a desired future state (target eigenvalues), find the present actions (graph modifications) that produce it.

**Do now:** **PUBLISH as a standalone web app.** It's already deployed on Cloudflare Workers. The reverse-causality framing is genuinely novel in productivity tools. Could integrate with the fleet as an agent-level planning tool — each agent reverse-actualizes from its conservation target.

### 10. **Hebbian Router** (2026-05-22) — Neurons That Fire Together, Wire Together

This was built in parallel with the spectral framework but implements the same principle: structure emerges from use, not design.

**Connects to:** Fiedler vector routing (the spectral equivalent), the fleet's agent composition, conservation as "what persists." The Hebbian router IS the time-domain version of the Laplacian's structural analysis.

**Do now:** **UNIFY with the spectral framework.** Hebbian routing = temporal conservation. Spectral routing = structural conservation. The combination — a router that uses BOTH Hebbian weights AND Fiedler projections — would be genuinely novel. The fast path + chaos exploration maps directly to flow-state architecture.

### **triplet-miner** (2026-05-22) — Contrastive Learning from Git History
Mining (anchor, positive, negative) triplets from git commit messages and diffs for contrastive learning. The hard-negative and semi-hard strategies with 3-shingle Jaccard similarity are production-quality. This is how you train models to understand CODE INTENT from git history.

**Do now:** Train an embedding model on the fleet's git history. The resulting vectors would be the semantic fingerprints that the spectral framework needs for agent-to-agent alignment scoring.

### **vector-novelty** (2026-05-22) — O(n) Diversity Scoring
Centroid-based novelty scoring for agent populations — O(n) instead of O(n²) pairwise comparison. With 128-dimensional latent vectors and NumPy vectorization, it processes 1000 agents in ~2ms.

**Do now:** This IS the diversity metric for the fleet. Integrate with conservation scoring — conservation × novelty = the true fitness function for agent evolution.

### **quality-gate-stream** (2026-05-07) — Tile Quality as a Science
The 5-dimension tile scoring system (length × technical depth × specificity × structure × question quality) with weighted scoring and mark-and-sweep GC is the quality infrastructure that makes PLATO tiles trustworthy.
