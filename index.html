A Python SDK for agents with persistent memory

SuperInstance is a Python SDK for building agents that remember across sessions. Agents store memory as markdown files on disk, can be organized into in-memory fleets, and optionally route questions through a DeepInfra LLM. The current codebase is a small, tested SDK — not a distributed system.

γ + η = C
γ — generation cost (tokens, time, API calls)
η — innovation value (new code, tests, discoveries)
C — total budget, conserved across all agents

The fleet design uses a guiding concept: generation cost (γ) and innovation value (η) trade off against a fixed budget (C). This is a design heuristic, not a measured or enforced law. No code in the current SDK implements this constraint.

What the SDK does today: agents with persistent markdown memory, an in-memory fleet for coordination, a thread-safe LRU cache for instance reuse, and optional DeepInfra LLM integration. Install with pip install superinstance.

The current SDK is small and tested. The distributed fleet architecture described in the roadmap is not implemented yet.— See the README for details

What the SDK Provides

🧠 Persistent Agent Memory

Agents store facts as markdown files (SOUL.md, MEMORY.md, diary entries) and recall them by keyword. Memory persists across process restarts. Optional DeepInfra LLM integration for reasoning over stored context.

from superinstance import Agent

agent = Agent("researcher")
agent.remember("Prefers Python", category="preference")
print(agent.recall("Python"))

Fleet Coordination

Create agents with tags, broadcast messages to the fleet, and dispatch tasks to tagged subsets. The fleet registry is in-memory — no external broker required.

from superinstance import Fleet

fleet = Fleet("team")
fleet.create_agent("scout", tags=["research"])
fleet.create_agent("writer", tags=["content"])
fleet.broadcast("New project started")

⚡ Instance Reuse Cache

A thread-safe LRU cache reuses agent instances across calls with configurable TTL eviction. Requesting the same agent twice returns the same object.

from superinstance import get_agent

a1 = get_agent("coder")
a2 = get_agent("coder")
assert a1 is a2  # same instance

How It Works

Agents with Memory
Each agent stores facts as markdown files on disk — SOUL.md, USER.md, MEMORY.md, and a dated diary. Memory persists across process restarts and can be recalled by keyword.
In-Memory Fleet
Agents can be organized into fleets with tags. Broadcast messages to all members or dispatch to a tagged subset. The registry is in-process — no external broker.
Instance Caching
A thread-safe LRU cache reuses agent instances with configurable TTL eviction. Requesting the same agent name twice returns the same object.
Optional LLM
When DEEPINFRA_API_KEY is set, agent.ask() routes questions through a DeepInfra LLM with memory context. Without it, falls back to keyword search over stored memories.
